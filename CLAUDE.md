## Project: AI Tools Expense Tracker

> Full architecture, data flow, and dev rules are in [AGENTS.md](./AGENTS.md) — read it first.

Python + Next.js expense tracker. Scans Gmail for invoices, builds Excel + Next.js dashboard, deploys to GitHub Pages.

### Commands
```bash
# Python
python build_report.py          # build Excel + docs/data.json
python auto_invoice.py          # run Gmail invoice scan manually

# Dashboard (Next.js)
cd dashboard-web && npm run dev    # dev server
cd dashboard-web && npm run build  # prebuild: sync-data.mjs, then Next.js build
```

### Architecture
```
build_report.py          — merges all sources → Excel + docs/data.json
auto_invoice.py          — Gmail scanner (runs daily 08:00 via Task Scheduler)
manual_receipts.json     — manual entries (written via dashboard → GitHub API)
dashboard-web/           — Next.js dashboard (deployed to GitHub Pages via docs/)
docs/                    — GitHub Pages root (auto-generated, never edit manually)
```

### Key Gotchas
- Branch is `master` (not `main`) — use `git push origin master`
- Never edit `docs/data.json` directly — generated by `build_report.py`
- Windows absolute paths in Python must be guarded with `os.environ.get("CI")` check
- CapCut uses hardcoded fallback `CAPCUT_FALLBACK_ILS = 49.90` — update if price changes
- Adding a transaction to BOTH `MANUAL_TRANSACTIONS` and `manual_receipts.json` creates duplicates
- GitHub Actions triggers on push to `manual_receipts.json` → rebuilds everything automatically
